遍历DOM

得到一个元素节点后，可以使用如下5个属性来找到其他相关的元素。

（1）parentNode

该属性在HTML中找到包含该元素的元素节点（即父元素节点）。

（2）previousSibling 和 nextSibling

这两个属性找到当前节点的前一个节点或后一个节点（如果存在）。

（3）firstChild 和 lastChild

这两个属性找到当前元素的第一个或最后一个节点。

注意

  1. 这些属性是当前节点的属性（而非用于选择元素的方法）因此不会以()结尾。
  2. 如果使用了这些属性，但是它们不包括前/后一个兄弟节点，或是第一个/最后一个子节点，那么结果会是null。
  3. 这些属性是只读的，它们只能用来选择一个新节点，而不能改变其父亲节点或子节点。

遍历DOM可能会遇到一些麻烦，因为有些浏览器会在元素之间添加一个文本节点，不管它们之间是不是真的空白。

解决该问题的方法：

  1. 去掉空白（可以把关闭标签的尖括号和下一个开始标签的尖括号连在一起）。
  2. 避免使用这些DOM属性。
  3. jQuery方法。（因为这种浏览器差异性，才导致jQuery大受欢迎）。

获取和更新元素内容

文本节点

在从一个元素导航到它的文本节点后，使用如下属性：

nodeValue 访问节点文字

包含元素

当操作一个元素节点（而不是其文本节点时），这个元素可能包含标签。可以选择需要获取/更新标签，或是获取/更新本文。

innerHTML 获取/设置文本和标签 textContent 仅获取/设置文本 innerText 仅获取/设置文本

使用nodeValue获取和更新文本节点

<li id = "one"><em>fresh</em>figs</li>

下面代码展示了如何访问第二个文本节点：它会返回第二个文本节点figs

document.getElmentById("one").firstChild.nextSibling.nodeValue

如果需要访问元素中的文本，首先要获得元素节点，然后再获得其文本节点。 下面例子使用nodeValue属性来更新文本节点的内容。

var itemTwo = document.getElementById('two');
var elText = itemTwo.firstChild.nodeValue;
elText = elText.replace('pine nuts','kale');
itemTwo.firstChild.nodeValue = elText;

使用textContext(和innerText)获取和更新文本

使用textContext属性可以获取或更新包含元素（及其子元素）中的文本。

<li id="one" class="hot"><em>fresh</em>figs</li>

如果想要获得<li>元素的所有文本，同时忽略元素内的所有标签，可以在<li>包含元素上使用textContext属性。在本例中将返回fresh figs。

var text = document.getElementById("one").textContent;

var text1 = document.getElementById("one").innerText;

兼容性： chrome 都支持，ie8以下不支持。 innerText在 ie8中 输出为undefined

innerText 应该避免使用，原因是 （1）支持情况差：firefox不支持。因为innerText不属于任何标准。（2）遵从CSS，它不会返回任何被CSS隐藏的内容。（3）获取文本的速度比textContent更慢。

添加或移除HTML内容

有两种非常不同的方式来添加和移除DOM树中的内容：innerHTML属性以及DOM操作。

注意：此处是innerHTML的安全风险，一会再写。。。

使用方法： 1.创建变量以保存标签：

var item = '<em>Fresh</em>figs';

2.选中需要更新内容的元素

var node = document.getElementById("one");

3.使用新的标签更新选中元素的内容

node.innerHTML = item;

DOM操作很容易地针对DOM树中的独立节点，而innerHTML更适合用来更新整个片段。

用innerHTML来更新文本和标签：

var firstItem = document.getElementById("one");
var itemContent = firstItem.innerHTML;
firstItem.innerHTML = '<ahref=\"http://example.org\">'+itemContent+'</a>';

使用DOM操作添加元素

1.创建元素 createElement()

2.设置内容 createTextNode()

次方法会创建一个新的文本节点。同样，该节点需要被保存到变量中。可以使用appendChild()方法将该节点添加到元素节点中。

3.添加到DOM中 appendChild()

该方法允许指定需要把这个节点添加到哪个元素上，并作为该元素的子节点。

通过DOM操作移除元素

1.把需要移除的元素保存到一个变量中。

2.把该元素的父元素保存到变量中，通过parentNode属性。

3.把元素从它的包含元素中移除。 removeChild()方法。

技术比较

3种向web页面添加HTML的方法，做个比较：

  1. document.write()

document的write()方法可以很简单地向页面的源代码中添加内容。不过不推荐使用。

优点：

  • 可以快速、简单地让初学者理解何时向页面中添加内容。

缺点：

  • 只在页面出事化时加载时有效。
  • 如果在页面加载后使用该方法，则会碰到：
    1. 整个页面都被覆盖
    2. 不是想页面中添加内容
    3. 创建一个新的页面
  • 在严格验证的XHTML 中可能会遇到问题
  • 程序员通常不太喜欢这个方法，也很少使用。
  1. element.innerHTML

该属性允许以字符串的方式，获取/更新任何元素中的整个内容（包括里面的标签）。

优点：

  • 和DOM操作方法相比，可以使用更少的代码添加大量的新标签。
  • 向页面中添加大量新元素时，速度比DOM操作更快。
  • 当需要移除元素中的所有内容时，它更简单（直接设置一个空字符串即可）。
/*这是我之前写的重置二叉树的例子*/
addEventHandler(resBtn, "click", function() {
    tree = new BinarySearchTree();
    tree1.innerHTML = '';
});

缺点：

  • 不应该用它来添加来自用户输入的内容，因为可能会带来严重的安全隐患。（如xss攻击）
  • 在添加一个很大的DOM片段式，这个方法很难独立区分出每一个元素。
  • 事件处理程序可能不像预期那样生效。
  1. DOM操作

DOM操作提供了一组方法和属性，用来访问、创建以及更新元素和文本节点。

优点：

  • 如果DOM 片段中拥有大量的兄弟节点，处理其中一个元素节点时使用这种方法更合适。
  • 不会影响事件处理程序。
  • 可以轻易的使用脚本来逐步添加元素。

缺点：

  • 如果需要对页面内容进行大量修改，它的速度比innerHTML属性更慢。
  • 与使用innerHTML相比，它需要更多的代码来实现同样的功能。

属性节点

得到一个元素节点后，可以再这个元素上使用其他一些对象属性和方法来获取和修改它的HTML属性。

获取和更新属性涉及两个步骤：

  • 首先，选中拥有该属性的元素节点，然后在后面加上一个点号。
  • 然后使用如下方法或对象属性来操作元素的HTML属性。

方法

  1. getAttribute() 获取属性值
  2. hasAttribute() 检查元素节点是否包含特定属性
  3. setAttribute() 设置属性
  4. removeAttribute() 从元素节点移除属性

属性 className 获取或设置class属性的值 id 获取或设置id属性的值

创建属性并修改其值

使用className 这个对象属性可以修改class这个HTML属性的值。如果class属性不存在，则会创建一个，并赋予它特定的值。

当存在一个对象属性时，通常认为直接给这个对象赋值要优于使用setAttribute()方法。

注意：这两个方法都会覆盖class属性的整个值，并不会把新值添加到class属性已有的值中。

如果需要向class属性的已有值中添加新值，需要先获取属性的值，然后把新值得文本追加到已有的属性值(或者使用jQuery的.addClass()方法)。
